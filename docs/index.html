The Problem

Managing seasonal decorations at scale is harder than it looks.

✕ Before

  • 200+ items scattered across totes, boxes, and zones — no single source of truth
  • Manual tracking of what's deployed vs packed vs in repair
  • No record of which outlets, cords, or zones power which displays
  • Season transitions were error-prone and time-consuming
  • No visibility into maintenance history or spend over time

✓ After

  • Real-time inventory across 10+ specialized subsystems with full audit trail
  • Automated status transitions triggered by packing, deployments, and maintenance events
  • Power network graph tracking outlets → cords → items
  • Full seasonal lifecycle: plan → stage → deploy → teardown → archive
  • AI agents for natural language queries, data enrichment, and receipt scanning

Platform

13 Subsystems

Each subsystem is an independent unit with its own Lambda handler(s), DynamoDB table(s), and API routes.

📦 Items Core

Central inventory for 200+ decoration items. Full CRUD, 5 status states, cascade deletes across 4 tables, async AI-generated search text via Claude Haiku.

🗄️ Storage

Manages totes and shelving units. Storage units are stored as items with class=Storage in the shared table. Packing events flow async via EventBridge — no direct item writes.

🎃 Deployments

Full seasonal lifecycle: plan → stage → deploy → teardown → archive. Tracks zones, power connections, and item status transitions via composite DDB key patterns.

🔧 Maintenance

Repair and inspection lifecycle with reusable schedule templates. Writes to 3 tables per completion. Work window constraints enforce April–September scheduling.

📋 Audit

DynamoDB Streams consumer tracking changes across 8 sources. Normalized audit records with JSON diff visualization, 90-day TTL, and bulk CSV/JSON export.

🖼️ Images

Centralized photo system for the entire platform. Presigned S3 uploads, Sharp-based WebP optimization via EventBridge trigger, Object Lock on confirmed assets, 10+ GSI patterns.

💰 Finance

Cost and vendor tracking with Claude Vision receipt scanning. AI audit log for all extractions. Vendor metadata syncs to items via EventBridge. CDN-served ReceiptExtractorWidget web component.

🔍 Inspector AI

Automated data quality rules (6+ types) with Inspector Gadget — a Bedrock Haiku agent that enriches violations with context, resolution paths, and confirmation flags. Read-only by design.

💡 Ideas

Tracks decoration concepts from ideation through build. EventBridge → SQS → consumer aggregates build costs to Finance on completion. Async Claude Haiku search text generation.

🖼️ Gallery AI

Public seasonal photo showcase with Cohere Embed v4 vector similarity for AI-powered tag suggestions. Visitor uploads supported. Sparse GSI for public item queries — no auth required to browse.

🛠️ Workbench

Read-only seasonal dashboard aggregating active ideas and maintenance tasks. 3-column view by season with year selector and deep-links to source subsystems.

🎛️ Admin AI

Central hub aggregating system-wide stats and hosting the Iris AI assistant panel. System map shows all subsystem health and counts at a glance.

📌 Tracker

Developer-facing project management with epics, issues, and tasks. Drag-to-reorder priority ranking using float midpoint insertion. Tasks stored in parent issue maps — no separate records.

Intelligence

AI Features

Three AI models, four distinct integration patterns, zero ML infrastructure to manage.

Claude 3.5 Sonnet · Bedrock

Iris — Admin AI Assistant

Natural language interface embedded in the admin panel. Iris runs a 10-iteration agentic loop: it calls existing SpookyDecs API Gateway endpoints as tools, resolves item nicknames ("Zero", "the big reaper") to IDs via semantic search, and confirms before any write operation. Auth cookies are forwarded to downstream APIs — no separate auth layer needed for the agent.

search_items get_item pack_item create_maintenance_record create_item
Claude 3 Haiku · Bedrock

Inspector Gadget — Violation Enrichment Agent

When the Inspector subsystem flags a data quality violation, Inspector Gadget runs a read-only Bedrock Haiku agentic loop to gather context from items, maintenance records, images, storage, and change history. It writes agent_notes, resolution_path, and requires_confirmation back to the violation record. Skip-if-populated logic prevents redundant re-annotation.

get_item get_maintenance get_images get_audit_history
Claude Vision · External API

Finance Receipt Scanner

The Finance subsystem uploads receipt photos to S3 via presigned URL, then passes the image to Claude Vision for automatic line-item extraction. Results are written to the finance table and every AI extraction is logged to a dedicated ai_audit_log table for compliance. The extractor is packaged as a CDN web component (ReceiptExtractorWidget) served from assets.spookydecs.com.

Cohere Embed v4 · Vector Search

Gallery — AI Tag Suggestion

When a photo is uploaded to the Gallery, Cohere Embed v4 generates an embedding and compares it against a pre-built item embedding index stored in S3. The top-k most similar items are surfaced as tag suggestions — enabling semantic photo-to-item matching without a vector database. The index is built asynchronously and cached in S3.

⚡

Async Search Text Generation

Items and Ideas both generate a plain-English search_text attribute via Claude Haiku immediately after creation — asynchronously, outside the API response path. This powers natural language queries in Iris and keyword search across the inventory without a dedicated search service.

Engineering

Technical Highlights

⚡

Sub-50ms Edge Auth

Lambda@Edge validates Cognito JWTs at CloudFront edge nodes before requests reach the origin — no auth round-trip, no dedicated auth service.

🔄

Event-Driven Architecture

EventBridge + SQS decouple all inter-subsystem communication. Storage, Finance, Ideas, and Gallery all publish domain events consumed asynchronously.

🗄️

Single-Table DynamoDB Design

Composite PK+SK patterns, sparse GSIs, and DynamoDB Streams feed the audit trail across 8 event sources — all without a relational database.

📸

Automated Image Pipeline

S3 upload → EventBridge trigger → Sharp Lambda → WebP thumbnail. Object Lock prevents accidental deletion of confirmed photos. Presigned URLs bypass Lambda payload limits.

🚀

Full CI/CD Pipeline

CodePipeline + CodeBuild deploy frontend bundles automatically. SSM Parameter Store provides environment-specific config at build time — no secrets in code.

🌍

Multi-Environment Parity

Dev and prod mirror each other completely — separate CloudFront distributions, Cognito pools, API Gateway stages, and Lambda aliases. Shared asset CDN across both.

🧩

CDN Web Components

Shared UI components (header, upload modal, receipt extractor, API config) are served from assets.spookydecs.com and consumed via window.* globals — no build step required.

🔒

Least-Privilege IAM

Each Lambda has a scoped execution role. Secrets are stored in SSM Parameter Store and AWS Secrets Manager — never hardcoded or passed through environment variables in plaintext.

💸

Zero-Idle Cost Architecture

Serverless throughout: Lambda, DynamoDB on-demand, S3 lifecycle cleanup of temp uploads, and CloudFront edge caching reduce both origin load and data transfer costs.
